Ubuntu 安装配置 nginx ​

在 Ubuntu 上安装和配置 Nginx 的步骤如下：

1、更新软件包列表：打开终端，运行以下命令更新软件包列表：

shell
sudo apt update
1

2、安装 Nginx：运行以下命令安装 Nginx：

shell
sudo apt install nginx
1

3、启动 Nginx 服务：安装完成后，运行以下命令启动 Nginx 服务：

shell
sudo systemctl start nginx
1

4、设置 Nginx 开机自启：运行以下命令将 Nginx 设置为开机自启：

shell
sudo systemctl enable nginx
1

5、检查 Nginx 状态：运行以下命令检查 Nginx 的状态：

shell
sudo systemctl status nginx
1

如果 Nginx 正在运行，你将看到相应的状态信息。

6、 配置 Nginx：默认情况下，Nginx 的配置文件位于/etc/nginx/nginx.conf。你可以使用任何文本编辑器打开该文件进行配置。主要配置项包括 server、location 等，用于定义域名、端口号、文件路径等。根据你的需求进行配置。

7、 重启 Nginx 服务：完成配置后，运行以下命令重启 Nginx 服务以使配置生效：

shell
sudo systemctl restart nginx
1

现在，你已经成功在 Ubuntu 上安装和配置了 Nginx。你可以通过浏览器访问 Nginx 的默认网页，确认是否安装成功。

它的默认配置文件在/etc/nginx/sites-enabled下的default里面，html文件放置在/var/www/html目录

想看 nginx 配置的命令：

shell
nginx -t
1

nginx -t 是一个用于测试 Nginx 配置文件的命令。它会检查 Nginx 的配置文件是否有语法错误，并告诉您配置文件的解析状态。如果配置文件没有错误，您将看到一个 "syntax is ok" 的消息。如果有错误，您将看到一个 "test is successful" 的消息和具体的错误信息。

这个命令在修改 Nginx 配置文件后特别有用，因为您可以快速检查配置文件是否有错误，而不需要完全重启 Nginx。这对于开发和调试阶段特别有用，因为您可能经常需要修改 Nginx 配置文件。

卸载 nodejs 和 npm ​

在 Ubuntu 中卸载 Node.js 和 npm，可以通过以下步骤完成：

打开终端。

1、卸载 Node.js 和 npm：

shell
sudo apt remove --purge nodejs npm
1

这将卸载 Node.js 和 npm，并删除它们的配置文件。

2、如果你还想要删除 Node.js 和 npm 的安装目录和相关文件，可以执行以下命令：

shell
sudo rm -rf /usr/local/bin/npm /usr/local/bin/node /usr/local/lib/node_modules /tmp/npm*
1

这将删除 Node.js 和 npm 的安装目录和相关文件。请注意，这些命令可能需要一些时间来执行，具体时间取决于你的系统配置和安装的 Node.js 和 npm 的大小。

3、如果你还想要删除 Node.js 和 npm 的配置文件，可以执行以下命令：

shell
sudo rm -rf /etc/apt/sources.list.d/nodesource.list /etc/apt/preferences.d/nodejs-preferences
1

这将删除 Node.js 和 npm 的配置文件。请注意，这些命令可能需要管理员权限来执行。

完成以上步骤后，Node.js 和 npm 将被完全卸载。你可以通过输入以下命令来验证是否成功卸载：

4、验证 Node.js 是否卸载：

shell
node -v
1

如果没有显示版本号，说明 Node.js 已经被成功卸载。

验证 npm 是否卸载：

shell
npm -v
1

如果没有显示版本号，说明 npm 已经被成功卸载。

nohup 命令的用法与作用 ​

nohup命令是 Linux 和 Unix 系统中的一个命令，用于在终端退出后继续运行命令。它的全称为“no hang up”，意为“不挂起”。

用法：

nohup COMMAND [ARG ...] [&]

其中，COMMAND是要在后台运行的命令或脚本，ARG是可选的参数，&符号用于将命令放入后台运行。

作用：

nohup命令的主要作用是在终端退出后，使命令或脚本在后台继续执行。使用该命令可以在关闭终端或断开 SSH 连接后，继续运行需要长时间运行的进程或服务。

当使用 nohup 命令提交作业后，默认情况下，所有输出将被重定向到一个名为nohup.out的文件中。如果当前目录的nohup.out文件不可写，输出将被重定向到$HOME/nohup.out文件中。如果需要将输出重定向到其他文件或设备，可以使用重定向操作符（>）指定输出文件。

使用示例：

在后台运行一个命令，并将输出保存到文件中：

shell
nohup command > output.txt &
1

在后台运行一个长时间运行的进程或服务，并忽略挂起信号：

shell
nohup service &
1

需要注意的是，nohup 命令只是将命令放入后台执行，并不会自动重启或监控进程的状态。如果需要监控进程的状态，可以使用其他工具或脚本来实现。

nodejs 执行 shell 脚本 ​

在 Node.js 中执行 shell 脚本，可以使用 child_process 模块中的 exec 函数。这个函数可以运行一个 shell 命令，并返回一个可读流和可写流，以便读取命令的输出和向命令发送输入。

以下是一个简单的示例，演示如何使用 exec 函数执行一个 shell 脚本：

js
const { exec } = require("child_process");

exec("./path/to/your/script.sh", (error, stdout, stderr) => {
  if (error) {
    console.error(`执行出错: ${error}`);
    return;
  }

  console.log(`stdout: ${stdout}`);
  console.error(`stderr: ${stderr}`);
});

如果你要向 shell 脚本传递参数，可以将它们作为 exec 函数的第二个参数传递。例如：

  // ...

shell 脚本写入日志 ​

写日志 ​

在 Shell 脚本中，可以使用 echo 命令将输出重定向到日志文件。下面是一个简单的示例：

shell
#!/bin/bash

# 打开日志文件
exec &> /path/to/logfile.log

# 写入日志
echo "This is a log message"

# 关闭日志文件
exec 2>&-

注意，在执行完写入操作后，记得关闭日志文件。如果不关闭日志文件，它将继续写入日志消息，直到脚本执行完毕。

写入时间 ​

要在 Shell 脚本中向日志文件写入时间，您可以使用 date 命令来获取当前时间，并将其与您的日志消息一起写入文件。以下是一个示例：

shell
#!/bin/bash

# 打开日志文件
exec &> /path/to/logfile.log

# 写入日志，包括当前时间戳
echo "$(date): This is a log message"

# 关闭日志文件
exec 2>&-
